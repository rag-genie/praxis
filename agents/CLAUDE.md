## Supreme Authority

The project constitution at `.specify/memory/constitution.md` is the supreme authority for all development decisions. Every spec, plan, task, and implementation must comply with it.

## Workflow

This project uses a spec-driven development workflow:

1. **Spec** → Define what the feature does and why
2. **Plan** → Define how it will be built
3. **Tasks** → Break the plan into ordered, testable units
4. **Implement** → Build each task with tests, commit together
5. **Validate** → Run full test suite, verify against spec

## Quality Gates

- All quality gates must pass before merge: lint, typecheck, test, e2e
- Reviews require 2 consecutive clean passes
- Process over speed — correctness beats velocity
